Etapa 2: git log, git clone, .gitignore

¿Qué hace git log?

git log es el comando que te muestra el historial completo de commits de tu repositorio. Es como abrir el libro de registros donde quedó grabado todo lo que pasó en el proyecto desde el principio.

Cuando se ejecuta dicho comando Git responde con una lista de todos los commits en orden del más reciente al más antiguo.

¿Qué información muestra cada commit?

Cada commit en el historial muestra 4 datos:

1. Hash
commit fa6ce35b2d1234abcd...

El código único que identifica ese commit. Cuando haces un commit aprece unos numeros al costado ese es el hash.

2. Autor
autor: ¿Quién-hizo-el-commit? < correo que se configuro al inicio>

3. Fecha
Date: Fri May 16 15:10:00 2026
Cuándo se hizo el commit.

4. Mensaje
El mensaje que se escribio cuando se hizo el commit.

¿Por qué es importante git log?

Porque te permite ver toda la historial del proyecto, entender qué cambió en cada momento, quién lo cambio y por qué. En equipos de trabajo es fundamental para saber qué hizo cada persona.

Sus formas de uso

Forma completa - muestra todos los detalles:
git log

Forma resumida - muestra solo el hash corto y el mensaje, más fácil de leer:
git log --oneline
Esta es la forma más usada en el día a día porque es más limpia y rápida de leer.
